Features

WGS is a complete, automated ecosystem for running professional-grade dedicated servers on Windows — without the headache.

Core

🖥️ Easy Setup & No-CLI Operation

Download, extract, and run. WGS installs SteamCMD automatically and handles all server installations from the GUI. No PowerShell scripts, no manual config file editing — everything is point-and-click.

⚡ One-Click Install, Update & Maintenance

WGS automates server installation and updates via SteamCMD. Schedule automatic update checks, set crash-recovery restart policies, and let WGS keep your servers current without manual intervention.

🛡️ Security Built-In

WGS manages Windows Firewall rules automatically — opening ports on start, closing them on stop. All sensitive data (RCON passwords, Steam credentials, API tokens) is stored using Windows DPAPI encryption. Never stored as plain text.

🔧 Plugin Creator Wizard

Your game isn't natively supported? No problem. The graphical Plugin Creator lets you add any dedicated server executable in minutes:

  • Define game ports, launch arguments, and protocols visually
  • Set up RCON, Steam query, or custom health checks
  • Export and share plugins as .cs files
  • No coding or config-file knowledge required

📊 Real-Time Dashboard

Monitor CPU and RAM consumption per server, track active player counts, and review crash history with performance graphs — all from one central dashboard.

💾 Automatic Backups

Scheduled, compressed world-save backups with configurable retention policies. Your community data is backed up automatically before updates, keeping it safe no matter what.

📁 Integrated File Browser v1.1.0

Browse, view, and edit server configuration files directly within WGS. Upload files from your PC, create folders, rename and delete — without leaving the application.

⏰ Scheduled Tasks

Automate recurring actions: schedule restarts, update checks, backups, or custom console commands on a time-based schedule. Set it once, forget it.

Remote Control

🌐 Web Dashboard & REST API v1.1.0

WGS includes a built-in HTTP server accessible from any browser — including mobile. Enable it in Settings → Web Remote Control and navigate to http://your-ip:8765/ui.

  • Real-time server status, CPU/RAM, player count
  • Start, stop, restart servers from the browser
  • Live console log viewer and command input
  • Full REST API for scripting and automation
  • Token-based authentication with role support

→ REST API Reference

🤖 Discord Bot

Control your servers from any Discord channel. The bot supports start, stop, restart, update, backup, and console commands. Configure allowed user IDs and a custom command prefix for full control over who can do what.

→ Discord Bot Setup

🔔 Discord Webhooks

Get notified on start, stop, crash, and update events in any Discord channel. Set a global webhook in Settings, or override it per server for separate notification channels per game.

🖥️ Multi-Machine Management v1.1.0

Run WGS on multiple machines and connect them together. A master WGS instance can monitor and control slave instances across your local network or the internet — all from a single panel. Each slave uses its own Web API token for authentication.

New in v1.1.0

📈 Bandwidth Monitoring v1.1.0

Per-server real-time network traffic graphs showing upload and download usage. Track bandwidth consumption over time to understand your hosting costs and detect unusual activity.

👥 User Management v1.1.0

Full multi-user access control for the web interface and REST API:

  • Admin role — full control, including start/stop/update
  • Viewer role — read-only access to stats and logs
  • Each user has an individual API token
  • Full audit log of all user actions (login, start, stop, etc.)

📦 Steam Workshop Integration v1.1.0

Manage mods for your servers directly from WGS:

  • Search the Steam Workshop by name or ID from within the app
  • Install and track mods per server instance
  • One-click update all mods to latest versions

🌐 UPnP Automatic Port Forwarding v1.1.0

WGS can automatically open and close port mappings on your router — no manual configuration needed.

  • Requests port mappings from the router when a server starts
  • Removes the mappings automatically when the server stops
  • Works with any UPnP-capable router (most home routers)
  • Test button in Settings verifies router compatibility instantly
Warning
UPnP automatically modifies your router's firewall rules without confirmation. Only enable this if you understand the risks and fully trust your network environment.

📋 Server Templates v1.1.0

Save current server settings as a reusable template. Export templates as JSON files to share configurations between machines or restore them after reinstall. Import templates to spin up identical server setups in seconds.

🏆 Player Statistics v1.1.0

Session-level tracking stored in SQLite: join time, leave time, and total playtime per player per server. View stats per server directly in the WGS interface.

New in v1.2.0

⧉ Server Cloning v1.2.0

Duplicate any server with all its settings in one click. WGS automatically assigns conflict-free ports for the clone so you can spin up a second instance of any game immediately.

💤 Wake-on-Demand v1.2.0

The server starts automatically the moment the first player tries to connect, and stays off when no one is playing. WGS listens on the game port for an incoming connection and brings the server up transparently — saving CPU, RAM and power when idle.

📧 Email Notifications (SMTP) v1.2.0

Receive start, stop, crash, and update alerts by email. Configure SMTP server, port, TLS and credentials per server in Settings → Notifications. Works with Gmail, Outlook, or any SMTP provider.

🧠 RAM Limiting v1.2.0

Set a hard memory cap per server using Windows Job Objects. If the server process exceeds the limit, WGS kills and restarts it automatically — preventing one runaway server from starving the whole machine.

📉 Historical Performance Charts v1.2.0

CPU and RAM history graphs with up to 1 hour of data per server, with a selectable time range (5 min / 15 min / 1 hr). Useful for spotting memory leaks, crash patterns, and peak usage times.

🔁 Crash-Loop Protection & Daily Restart v1.2.0

Configurable grace period between crash-restarts prevents a broken server from cycling indefinitely. Pair it with a daily scheduled restart to keep servers fresh and memory usage low.

New in v1.2.3

😴 Shut Down When Empty v1.2.3

The companion to Wake-on-Demand. WGS monitors the player count while a server runs and stops it automatically after a configurable idle timeout (default 10 min) when all players have left. Together they create a full on-demand server lifecycle — zero idle resource cost.

🔔 Per-Server Discord Webhooks v1.2.3

Each server can now have its own Discord webhook URL in Settings → Discord alerts. Use it to route notifications for different games to different Discord channels. Falls back to the global webhook if left empty.

📂 Backup Restore via Web Dashboard v1.2.3

A Backups button on each server card in the web GUI lists all available backups and lets you restore with one click — no need to open WGS on the host machine.

📁 Selective Backup Paths v1.2.3

Back up only the save folder instead of the entire game install — dramatically reducing backup size for large games. Built-in defaults for Enshrouded, Palworld, ARK, DayZ, V Rising, Satisfactory, Project Zomboid and Terraria. Fully customisable per server.
