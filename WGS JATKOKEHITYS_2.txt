1. Älykäs Steam-pelinimen tunnistus Workshop-linkeistä (Mod Name Resolver)
Mitä se tekee?
Kun admin lisää uuden modin syöttämällä pelkän ID-numeron (esim. 221100 DayZ-modille), WGS ei näytä käyttöliittymässä pelkkää kuivaa numeroa. Järjestelmä tekee taustalla kevyen HTTP-kutsun Steamin julkiseen Web-API-rajapintaan, hakee sieltä modin oikean nimen (esim. "Namalsk Island") ja tallentaa sen SQLite-tietokannan ServerMods-taulun ModName-sarakkeeseen. Tämä nimi näytetään sitten siististi WPF:n käyttöliittymässä.

Context: We are building a Generic Steam Workshop Mod Manager for the Windows Game Server (WGS) application (.NET 8 / WPF / MVVM).
Task: Implement a helper service inside WGS that resolves a Steam Workshop Item ID into its actual human-readable title.

Requirements:
1. Create a method (e.g., in SteamWorkshopService) that queries the public Steam Web API:
   URL: https://api.steampowered.com/ISteamRemoteStorage/GetPublishedFileDetails/v1/
   Method: POST (with "itemcount=1" and "publishedfileids[0]={ModId}" as FormUrlEncodedContent).
2. Parse the JSON response to extract the "title" fields from "response.publishedfiledetails[0]".
3. If the API call fails or the network is down, gracefully fall back to using the "ModId" string as the name.
4. Integrate this into the mod installation flow: when InstallOrUpdateModAsync is called, resolve the name first before saving the record into the SQLite 'ServerMods' database table (ModName column).
5. Ensure the network call is non-blocking (async/await) and properly error-handled.


2. Varmuuskopioiden automaattinen siivous (Backup Retention Policy)
Mitä se tekee?
WGS ottaa automaattisesti .zip-varmuuskopion palvelimen tallennuksesta ennen jokaista päivitystä. Tämä toiminto estää kiintolevyn täyttymisen ja palvelimen kaatumisen (Low Disk Space). Se tarkistaa varmuuskopiokansion sisällön aina uuden backupin luonnin jälkeen, järjestää ne päivämäärän mukaan ja poistaa automaattisesti vanhimmat tiedostot, jos asetettu raja ylittyy.


Context: WGS handles server backups via BackupService by creating .zip files of world saves. We need to implement a retention policy to prevent the hard drive from filling up.
Task: Extend BackupService to automatically clean up old backup files based on a configurable limit.

Requirements:
1. Add a configuration property to the Server Settings model: 'MaxBackupCount' (int, default to 5 or 10).
2. Inside BackupService, after a new backup .zip is successfully created, scan the backup directory for that specific server instance.
3. Get all files matching the backup naming pattern, and order them by creation date (oldest first).
4. If the total number of backup files exceeds 'MaxBackupCount', delete the oldest files until the count matches the allowed limit.
5. Add try-catch blocks around File.Delete to ensure that if a backup file is locked, it won't crash the entire update loop, but logs a warning instead.



3. Valmiit palvelinpohjat (Config Presets)
Mitä se tekee?
Kun käyttäjä asentaa uuden palvelimen, peli luo oletuksena tyhjän tai hyvin perusmuotoisen asetustiedoston (kuten server.cfg tai ping.ini). Tämä toiminto tuo WGS:ään valmiita asetuspohjia (esim. "DayZ PvE No-Raiding", "Rust 5x Loot"). Kun admin valitsee pohjan, WGS osaa lukea kyseisen pelin asetustiedoston ja muokata tai ylikirjoittaa tekstirivit valmiilla arvoilla, jolloin adminin ei tarvitse osata vaikeita komentorivejä tai muuttujia ulkoa.


Context: WGS has a ConfigEditorService to modify server configuration files. We want to add a "Config Presets" feature so admins can apply pre-made gameplay templates (e.g., Vanilla, High-Loot, PvE-Only) with one click.
Task: Implement a text-replacement preset system within ConfigEditorService.

Requirements:
1. Create a folder inside the application root called 'Presets'. Inside, store game-specific configuration templates (e.g., 'dayz_pve.cfg', 'rust_5x.cfg').
2. In the UI (ServerDetailView), add a ComboBox listing available presets based on the active server's game type.
3. When the admin selects a preset and clicks "Apply Preset", WGS should read the target server's active config file.
4. The service will merge or overwrite the key-value pairs in the server's config file with the ones defined in the preset file.
5. Create a safe verification prompt in WPF ("Are you sure you want to overwrite your current configuration with this preset?") before writing changes to the disk, and automatically trigger a backup of the old config file before replacing it.



3. Automaattinen RAM-muistin vapautus ennen palvelimen käynnistystä (Trim Working Set)
Mitä se tekee?
Kun Windows-koneella pyöritetään useita pelipalvelimia tai jos koneella tehdään muuta samaan aikaan, Windows jättää usein taustalle "roikkumaan" muistia, jota suljetut ohjelmat ovat käyttäneet. Tämä toiminto pakottaa Windowsin siivoamaan turhan muistin juuri silloin, kun uusi pelipalvelin on käynnistymässä, jotta peli saa mahdollisimman paljon puhdasta RAM-muistia käyttöönsä.

Miten se toimii?
Käytetään Windowsin omaa API-kutsua (EmptyWorkingSet), jolla tyhjennetään käyttöjärjestelmän välimuistia ja muiden prosessien käyttämätöntä muistia levylle (pagefile), vapauttaen fyysistä RAM-muistia pelille.


Context: Game servers require large amounts of physical RAM upon startup. We want to implement a minor system optimization helper before launching a server executable.
Task: Create a RAM optimization utility method inside SystemMetricsService or ServerManagerService.

Requirements:
1. Use Windows Native API (P/Invoke) to call 'EmptyWorkingSet' from 'psapi.dll'.
2. Create a method 'OptimizeSystemMemory()' that iterates through non-critical running background processes and flushes their working set memory if safe, or calls the garbage collector explicitly.
3. Trigger this method inside ServerManagerService right before a new game server process is initiated (Process.Start()).
4. Add a checkbox in global WGS settings: "Optimize system RAM before starting a server" to allow users to toggle this behavior on/off.



4. "Pikaliittyminen" – Steam Connect -linkki UI-ikkunassa (Direct Connect Button)
Mitä se tekee?
Kun palvelin on saanut ladattua maailman pystyyn, adminin tai hänen kavereidensa ei tarvitse avata peliä, odottaa valikoita, kirjoittaa IP-osoitteita ja etsiä palvelinta listalta. WGS:n käyttöliittymään tulee yksi painike, jota klikkaamalla peli käynnistyy tietokoneella ja yhdistää suoraan kyseiselle palvelimelle.

Context: We want to add a "Join Server" convenience feature to the WGS UI.
Task: Implement a button in ServerDetailView that launches the game client via Steam and connects directly to the server's IP and Query Port.

Requirements:
1. Add a button "Join Server" and another button "Copy Connection Link" next to the server status indicator.
2. When "Join Server" is clicked, use Process.Start to execute the 'steam://connect/{IP}:{QueryPort}' URI protocol.
3. Ensure it runs safely using ProcessStartInfo with UseShellExecute = true.
4. When "Copy Connection Link" is clicked, copy the exact 'steam://connect/{IP}:{QueryPort}' string to the Windows Clipboard so the admin can easily share it with friends on Discord.
